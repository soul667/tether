## 项目概述

Tether 是一个机器人自主功能性操作（Autonomous Functional Play）系统，核心思想是：

1. **轨迹变形（Trajectory Warping）**：从少量演示（≤10 个）中，通过语义关键点对应关系将动作迁移到新场景
2. **自主循环（Autonomous Play）**：利用视觉语言模型（VLM）驱动"任务选择→执行→评估→改进"的持续循环
3. **数据积累**：通过自主探索不断积累高质量轨迹数据，用于训练闭环模仿策略

## 核心架构

系统由四个层次组成：

### 1. 编排层（Orchestration）

**`runner.py`** — 系统核心，`Runner` 类负责整个流程的协调：

```
Runner
├── init_vlm()          → 初始化两个 Gemini VLM 实例
├── prepare_bootstrap() → 预处理演示数据到动作库
├── run_single()        → 单任务模式：选定动作反复执行
└── run_cycle()         → 自主循环模式：
    ├── collect_scene_image()          # 捕获当前场景
    ├── generate_action_plan()         # VLM 规划任务
    ├── warp_trajectory()              # 变形轨迹
    │   ├── run_correspondence()       # 计算对应关系
    │   └── warp_trajectory()          # 执行变形
    ├── send_trajectory()              # 发送到机器人
    ├── query_gemini_evaluate_success()# 评估是否成功
    └── ucb.update_arm()               # 更新探索策略
```

### 2. 计算层（Computation）

- **`run_correspondence.py`** — 通过 GeoAware 和 MASt3R 计算演示帧与目标场景间的像素级语义对应关系，并通过立体三角测量获取三维位置偏移
- **`warp_trajectory.py`** — 基于关键点对应关系对演示轨迹进行平滑插值变形，保持速度一致性

### 3. 决策层（Decision Making）

- **`query_gemini.py`** — 封装对 Gemini VLM 的两种查询：
  - **动作规划**：给定场景图像，生成执行计划
  - **成功评估**：对比执行前后的图像，判断任务是否完成
- **`ucb.py`** — UCB（Upper Confidence Bound）多臂老虎机算法，平衡探索与利用来选择最优演示

### 4. 服务层（Services）

- **`serve_geo_aware.py`** — 将 GeoAware 语义对应模型封装为 RPC 服务（端口 50011）
- **`serve_mast3r.py`** — 将 MASt3R 三维匹配模型封装为 RPC 服务（端口 50022）
- 两者均使用 Python `multiprocessing.managers` 实现进程间通信

## 执行流程

### 自主循环模式（`mode=cycle`）完整流程

```
初始化
│
├─ 加载演示数据 (data_real/demos/)
├─ 预处理每个演示:
│   ├─ 转换轨迹格式 → trajectory_demo.npy
│   ├─ 提取关键点 → gripper_keypoints.npy
│   └─ 生成标注视频 → annotations/
├─ 构建动作库 (action_library.json)
│
循环开始 ─────────────────────────────────────
│
├─ 1. 捕获场景 → ZeroRPC → 保存图像
│
├─ 2. VLM 规划 → Gemini API
│   └─ 输入: 场景图像 + 可用动作列表
│   └─ 输出: 按优先级排序的动作序列
│
├─ 3. 对每个待执行的动作:
│   ├─ UCB 选择最佳演示
│   │
│   ├─ 计算对应关系:
│   │   ├─ GeoAware (localhost:50011): 语义特征匹配
│   │   ├─ MASt3R  (localhost:50022): 三维一致性验证
│   │   └─ 立体三角测量 → 三维位置偏移
│   │
│   ├─ 轨迹变形:
│   │   ├─ 按关键点分段变形
│   │   ├─ 位置: 线性 + 三次样条插值
│   │   ├─ 姿态: 球面线性插值 (SLERP)
│   │   └─ 速度归一化
│   │
│   ├─ 发送到机器人执行
│   │
│   └─ VLM 评估 → Gemini API
│       └─ 对比执行前后图像判断是否成功
│
├─ 4. 更新 UCB 状态
│
└─ 5. 定期重建动作库 ──→ 回到循环开始
```

```
┌─────────────────────────────────────────────────┐
│                 宿主机 / Docker                    │
│                                                   │

│  └──┬───┬───┬──┘                                 │
│     │   │   │                                    │

│     │   │                                        │
│     │   └── RPC ──→ MASt3R  (localhost:50022)    │
│     │                [GPU, conda: mast3r]        │
│     │                                            │
│     └──── RPC ──→ GeoAware (localhost:50011)     │
│                    [GPU, conda: geoaware]        │

└─────────────────────────────────────────────────┘
```

## 数据格式说明

### 轨迹格式

```python
# trajectory.npz 中的 "state" 数组
# 形状: (T, 7)  T = 时间步数
# 列:   [x, y, z, roll, pitch, yaw, gripper_state]
#        ──位置──  ───姿态(欧拉角)───   0=闭合/1=张开
```

### 相机标定文件

```json
{
    "{camera_id}_left": {
        "extrinsics": [x, y, z, roll, pitch, yaw],
        "intrinsics": [fx, fy, cx, cy, ...]
    }
}
```

### 对应关系结果

```json
{
    "0": {
        "position_delta": [Δx, Δy, Δz],
        "orientation_delta": [0, 0, 0],
        "infos": {
            "0": {
                "correspondence_score": 0.85,
                "distance_score": 0.92,
                "score": 0.88
            }
        }
    }
}
```

### Hydra 配置

系统使用 [Hydra](https://hydra.cc/) 管理配置，支持命令行覆盖：

```bash
# 覆盖运行模式
python runner.py mode=single action=bowl-table-shelf

# 覆盖任意参数
python runner.py num_warp_attempts=5 vlm.api_key_smart=xxx
```
