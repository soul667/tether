## 核心函数

### runner.py — 流程编排

| 函数 | 说明 |
|------|------|
| `Runner.__init__(cfg)` | 初始化 VLM、动作库、UCB 策略和 W&B 追踪器 |
| `Runner.init_vlm()` | 创建两个 Gemini 实例：smart（带思维链推理）和 fast（快速响应） |
| `Runner.prepare_bootstrap()` | 预处理所有演示数据：转换轨迹、提取关键点、生成标注视频、构建动作库 |
| `Runner.warp_trajectory(demo_dir, scene_dir, output_dir)` | 核心管线：调用 `run_correspondence` 计算对应 → 调用 `warp_trajectory` 执行变形 |
| `Runner.generate_action_plan(scene_dir)` | 使用多个 VLM 规划器投票选出最优动作序列 |
| `Runner.process_rollout(rollout_dir, action)` | 处理执行结果：评估成功、更新 UCB、生成可视化 |
| `Runner.run_single()` | 单任务模式：对指定动作反复执行（随机选演示） |
| `Runner.run_cycle()` | 自主循环模式：规划→执行→评估的完整闭环 |

**关键实现细节：**

- `generate_action_plan` 使用 `num_planners`（默认 2）个 VLM 并行规划，通过投票机制提高决策鲁棒性
- `warp_trajectory` 支持 `num_warp_attempts`（默认 3）次尝试，选择得分最高的变形结果
- 周期性重建动作库（`library_rebuild_interval` 控制频率）

### run_correspondence.py — 对应关系计算

| 函数 | 说明 |
|------|------|
| `run_correspondence(cfg, keypoint_indices, demo_dir, scene_dir, output_dir)` | 主函数：计算演示与目标场景间的语义关键点对应关系 |
| `load_geo_aware(port)` | 连接到 GeoAware RPC 服务端 |
| `load_mast3r()` | 连接到 MASt3R RPC 服务端 |
| `create_correspondence_visualization(...)` | 生成对应关系可视化图像（匹配点连线） |
| `create_triangulation_visualization(...)` | 生成三维三角测量可视化图像 |

**算法流程：**

1. 对每个关键点，从演示帧中提取其二维像素位置
2. 使用 **GeoAware** 在目标场景中寻找语义最相似的点（基于余弦相似度）
3. 使用 **MASt3R** 进行跨视角验证，确保三维一致性
4. 通过多视角 **立体三角测量** 计算三维位置
5. 基于综合评分（对应分数 + 距离分数）选择最佳匹配

### warp_trajectory.py — 轨迹变形

| 函数 | 说明 |
|------|------|
| `warp_trajectory(cfg, demo_dir, output_dir)` | 主函数：基于对应关系偏移量变形轨迹 |
| `warp_segment(segment, T_start, T_end)` | 对轨迹段进行插值变形（起始和结束变换矩阵间插值） |
| `normalize_warping_velocity(trajectory, ...)` | 重采样变形轨迹以保持速度一致性 |
| `linear_interpolate_xyz(positions)` | 使用三次样条插值平滑位置序列 |
| `spherical_interpolate_rpy(rotations)` | 使用球面线性插值（SLERP）平滑姿态序列 |
| `discretize_gripper(gripper_states, new_len)` | 智能重采样夹爪状态到新轨迹长度 |

**算法要点：**

- 将轨迹按关键点分段，每段独立变形
- 位置变形使用线性插值 + 三次样条平滑
- 姿态变形使用 SLERP 确保旋转平滑过渡
- 变形后进行速度归一化，防止末端执行器速度突变

### query_gemini.py — VLM 接口

| 函数 | 说明 |
|------|------|
| `query_gemini_plan_actions(cfg, vlm, target_task, action_list, scene_dir, save_dir)` | 向 Gemini 发送场景图像，获取动作规划方案 |
| `query_gemini_evaluate_success(cfg, vlm, trajectory_dir, action, keypoint_indices, output_dir)` | 向 Gemini 发送执行前后图像，评估任务是否成功完成 |

**使用的提示模板：**

- `prompts/plan_actions.txt` — 输入场景图像和可用动作列表，输出按优先级排序的动作方案
- `prompts/evaluate_success.txt` — 输入多视角执行前后图像，输出成功/失败判定

### ucb.py — 探索策略

| 函数 | 说明 |
|------|------|
| `UCB.select_arm(valid_arms)` | 基于 UCB 公式选择得分最高的演示，返回排序列表 |
| `UCB.add_arm(arm)` | 注册新演示到探索策略中 |
| `UCB.update_arm(arm, reward)` | 根据执行结果（0 或 1）更新统计 |
| `UCB.save_state() / load_state()` | 持久化探索状态到 JSON 文件 |

**UCB 公式：**

```
UCB = mean_reward + exploration × √(ln(t) / count)
```

- `mean_reward`：最近 `history_len`（默认 32）次执行的平均成功率
- `exploration`：探索系数（默认 0.5）
- `t`：总迭代次数
- `count`：该演示被选择的次数

## 服务端函数

### serve_geo_aware.py（端口 50011）

| 函数/方法 | 说明 |
|-----------|------|
| `GeoAware.get_processed_features(image)` | 提取图像的多尺度融合特征（Stable Diffusion + DINOv2） |
| `GeoAware.load_images(path)` | 加载并缓存图像特征（SHA256 缓存键） |
| `GeoAware.compute_correspondence(src, tgt, point)` | 基于余弦相似度在目标图像中找到最佳匹配点 |
| `serve_geo_aware(port=50011)` | 启动 RPC 服务 |

**模型架构：**
- 特征提取器：Stable Diffusion v1-5（扩散特征，尺度 3/4/5）+ DINOv2 ViT-B14
- 聚合网络：投影到 768 维统一特征空间
- 缓存机制：LRU 缓存，最多 10K 条目

### serve_mast3r.py（端口 50022）

| 函数/方法 | 说明 |
|-----------|------|
| `Mast3r.load_images(path1, path2)` | 对图像对执行不对称 MASt3R 匹配 |
| `Mast3r.compute_correspondence(src, tgt, point)` | 从已计算的匹配中查找最近邻点 |
| `serve_mast3r()` | 启动 RPC 服务（端口 50022） |

## 工具模块

### utils/vlm_utils.py — VLM 封装

- 封装 Google Gemini API 的调用逻辑
- 支持异步查询和文件上传
- 自动解析 JSON 格式的 VLM 回复
- 包含缓存和重试机制

### utils/robot_utils.py — 机器人通信

- `collect_scene_image()` — 通过 ZeroRPC 连接机器人采集场景图像
- `send_trajectory()` — 通过 rsync 发送轨迹数据到机器人并执行
- 默认连接地址：`172.16.0.1:4545`

### utils/geometry_utils.py — 三维几何

- 四元数 ↔ 欧拉角 ↔ 旋转矩阵互转
- 齐次变换矩阵操作
- 相机投影（世界坐标→像素坐标）
- 立体三角测量
- 姿态插值（SLERP）

### utils/calibration_utils.py — 相机标定

- 从 JSON 文件加载相机内参和外参
- 构建投影矩阵

### utils/annotation_utils.py — 图像标注

- 在图像上绘制轨迹曲线
- 标记关键点和夹爪状态
- 绘制极线

### utils/drawing_utils.py — OpenCV 绘图

- 坐标轴箭头绘制
- 夹爪几何形状绘制
- 半透明叠加绘制
- 文本标注

### utils/misc_utils.py — 杂项工具

- 轨迹文件 I/O（NPZ/NPY 格式）
- 位置修正和验证
- 视频处理（FFmpeg）
- 数据过滤

### utils/statistics_tracker.py — 指标追踪

- 线程安全的 W&B 指标记录
- 支持实验过程中的实时监控

### utils/timer_utils.py — 计时工具

- 装饰器和上下文管理器两种用法
- 用于性能分析和代码块计时

## Docker 与 Compose 使用

### Docker 镜像

```bash
# 构建
docker build -t tether:latest .

# 运行（指定任务）
docker run --rm --network host \
    -v ./data_real:/app/data_real \
    -v ./cache:/app/cache \
    -v ./conf:/app/conf \
    tether:latest \
    python runner.py mode=single action=bowl-table-shelf
```

### Docker Compose 服务

| 服务名 | 说明 | 激活方式 |
|--------|------|----------|
| `tether` | 默认服务（显示帮助） | `docker compose run --rm tether` |
| `tether-single` | 单次轨迹模式 | `ACTION=xxx docker compose --profile single run --rm tether-single` |
| `tether-cycle` | 自主循环模式 | `docker compose --profile cycle run --rm tether-cycle` |

所有服务均使用 `network_mode: host` 以访问宿主机上运行的 GeoAware、MASt3R 和机器人服务。

### Volume 挂载说明

| 挂载路径 | 说明 |
|----------|------|
| `./data_real:/app/data_real` | 演示数据和运行结果 |
| `./cache:/app/cache` | 特征缓存（加速重复计算） |
| `./conf:/app/conf` | 配置文件（可自定义覆盖） |
