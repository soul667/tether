## 前提条件

| 依赖项 | 说明 |
|--------|------|
| Docker | 安装 Docker Engine（建议 20.10+） |
| docker compose | 用于服务编排（Docker Desktop 自带） |
| GeoAware 服务 | 语义对应模型 RPC 服务（端口 50011） |
| MASt3R 服务 | 三维匹配模型 RPC 服务（端口 50022） |
| Gemini API Key | Google Gemini VLM 的 API 密钥 |
| 机器人硬件 | Eva Franka 机械臂或自定义机器人接口 |

> **注意**：GeoAware 和 MASt3R 需要 GPU 环境和独立的 conda 环境，不包含在本 Docker 镜像中。Docker 镜像仅打包了 Tether 主程序的运行环境。

## 第一步：构建 Docker 镜像

### 方式 A：使用脚本构建

```bash
./scripts/build.sh
```

### 方式 B：直接使用 Docker 命令

```bash
docker build -t tether:latest .
```

### 方式 C：使用 Docker Compose 构建

```bash
docker compose build tether
```

### 验证镜像

构建完成后验证环境是否正确：

```bash
./scripts/validate_env.sh
```

该脚本会在容器内导入所有 Python 依赖和项目模块，确认环境配置正确。

## 第二步：准备配置与数据

### 2.1 设置 API 密钥

编辑 `conf/config.yaml`，填入你的 Gemini API 密钥：

```yaml
vlm:
  api_key_smart: <你的_API_KEY>
  api_key_fast: <你的_API_KEY>
```

### 2.2 准备演示数据

将初始演示数据放在 `data_real/demos/` 目录下，每个演示的目录结构如下：

```
data_real/demos/{demo_name}/
  ├── trajectory.npz          # 机器人轨迹（状态为笛卡尔末端执行器位姿）
  ├── calibration.json        # 相机标定文件
  └── recordings/
      ├── {camera_name}.mp4   # 每个相机的录制视频
      └── frames/
          └── {camera_name}/
              ├── 00000.jpg
              ├── 00001.jpg
              └── ...
```

### 2.3 配置任务和相机参数

编辑 `conf/setting/real.yaml`：

```yaml
# 任务演示列表（键: 文件夹名, 值: 自然语言描述）
demo_names:
  bowl-table-shelf: "pick up bowl from table and place it on middle shelf"

# ZED 相机序列号
cameras:
  varied_camera_1: "你的相机序列号"

# 工作空间边界
oob_bounds:
  x: [0.2, 1]
  y: [-0.6, 0.6]
  z: [-0.1, 1]
```

## 第三步：启动外部服务

在运行 Tether 之前，需要在宿主机上分别启动以下服务：

### 3.1 启动 GeoAware 服务（端口 50011）

```bash
conda activate geoaware
python serve_geo_aware.py
# 等待显示 "Serving ..." 后继续
```

### 3.2 启动 MASt3R 服务（端口 50022）

```bash
conda activate mast3r
python serve_mast3r.py
# 等待显示 "Serving ..." 后继续
```

### 3.3 启动机器人服务

参考 [Eva 文档](https://github.com/willjhliang/eva) 启动 Eva 服务，或准备你自己的机器人控制接口。

## 第四步：运行 Tether

### 运行单次轨迹（Single Rollout）

使用脚本：
```bash
./scripts/run_single.sh bowl-table-shelf
```

或直接用 Docker：
```bash
docker run --rm --network host \
    -v ./data_real:/app/data_real \
    -v ./cache:/app/cache \
    -v ./conf:/app/conf \
    python runner.py mode=single action=bowl-table-shelf
```

输出保存在 `data_real/runs/<run_name>/rollouts_single/`。

### 运行自主循环（Autonomous Play）

使用脚本：
```bash
./scripts/run_cycle.sh
```

或直接用 Docker：
```bash
docker run --rm --network host \
    -v ./data_real:/app/data_real \
    -v ./cache:/app/cache \
    -v ./conf:/app/conf \
    tether:latest \
    python runner.py mode=cycle
```

输出保存在 `data_real/runs/<run_name>/rollouts_cycle/`。

## Docker Compose 用法

### 查看帮助
```bash
docker compose run --rm tether
```

### 运行单次轨迹
```bash
ACTION=bowl-table-shelf docker compose --profile single run --rm tether-single
```

### 运行自主循环
```bash
docker compose --profile cycle run --rm tether-cycle
```

## 常见问题排查

### Q: 容器无法连接 GeoAware/MASt3R 服务？

**A:** 确保使用了 `--network host` 模式（脚本和 docker-compose 已默认配置）。检查宿主机上的服务是否已启动并监听在 `localhost:50011` 和 `localhost:50022`。

### Q: 容器无法连接机器人？

**A:** 确保机器人控制服务已启动，且容器使用了 `--network host` 模式可以访问机器人 IP（默认 `172.16.0.1:4545`）。

### Q: 如何使用 GPU？

**A:** 本 Docker 镜像主要用于 Tether 主程序，不涉及 GPU 计算。需要 GPU 的 GeoAware 和 MASt3R 服务建议直接在宿主机的 conda 环境中运行。

### Q: 如何查看实验数据？

**A:** 运行时数据通过 volume 挂载与宿主机共享，可直接在 `data_real/runs/` 目录下查看：
- `rollouts_single/` 或 `rollouts_cycle/` — 执行结果
- `pipeline_*/correspondence/` — 对应匹配可视化
- `annotations/` — 轨迹标注视频
- `evaluation/` — VLM 评估记录
